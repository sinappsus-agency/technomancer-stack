# SOPs — Standard Operating Procedures

This directory contains the operational SOPs for a solo studio running the Technomancer stack.

These are product-ready operational documents — not templates. Each file is a working SOP you adapt to your context by replacing the `[BRACKET]` fields with your specifics.

## Format Standard

Every SOP in this directory follows this structure:

```
Title / Category / Version / Owner / Trigger / Frequency
OBJECTIVE
PREREQUISITES
STEPS (numbered, role + action + tool + output)
CHECKPOINTS (decision gates)
TOOLS REQUIRED
FAILURE MODES + ESCALATION
OUTPUT / HANDOFF
```

Owner = role, not person.
Steps = "You [do action] using [tool] → produces [output]"

---

## Product-Ready Standard

These SOPs are held to a higher bar than internal notes:

- Any contractor can execute from Day 1 without calling you
- All tool names are explicit
- Every step produces a named output
- At least two failure modes documented
- Written in second-person imperative

If a SOP cannot pass that test, it is a draft. Mark it `STATUS: DRAFT` at the top until it passes.
