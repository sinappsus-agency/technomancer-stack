# Security Checklist — Post-Deployment Verification

Run through this checklist after every new service deployment or server configuration change.

## UFW Firewall

```bash
# Check firewall status
sudo ufw status verbose

# Expected output: Status active
# Expected open ports: 22/tcp, 80/tcp, 443/tcp
# Everything else should be DENY
```

- [ ] Port 22 (SSH) open — consider restricting to your IP only
- [ ] Port 80 (HTTP) open — Traefik redirects to HTTPS
- [ ] Port 443 (HTTPS) open — all traffic
- [ ] All other ports DENY

## SSH Configuration

```bash
# Check SSH config
sudo grep -E "PasswordAuthentication|PermitRootLogin|PubkeyAuthentication" /etc/ssh/sshd_config
```

- [ ] `PasswordAuthentication no`
- [ ] `PermitRootLogin no` (or `prohibit-password` if you must)
- [ ] `PubkeyAuthentication yes`
- [ ] SSH key is installed in `~/.ssh/authorized_keys`

## Traefik Configuration

- [ ] Traefik dashboard is disabled OR behind basicAuth middleware
- [ ] All services behind HTTPS with valid Let's Encrypt certificates
- [ ] HTTP redirect to HTTPS enabled for all services
- [ ] Verify: `curl -I http://workflow.yourdomain.com` should return `301 Moved Permanently`

## Admin Interface Access

For each service, verify the admin URL requires authentication:

```bash
# Test each service URL - all should return redirect to login or 401
curl -I https://workflow.yourdomain.com/rest/login
curl -I https://s3-console.yourdomain.com
curl -I https://erp.yourdomain.com
curl -I https://vault.yourdomain.com
curl -I https://status.yourdomain.com
```

- [ ] n8n requires login
- [ ] MinIO Console requires login
- [ ] ERPNext requires login
- [ ] Vaultwarden requires login
- [ ] Uptime Kuma requires login (set during first run)

## Default Credentials

- [ ] All default passwords have been changed
- [ ] Admin passwords are 20+ characters, generated by password manager
- [ ] Passwords stored in Vaultwarden

## Docker Container Security

```bash
# Check no containers are running as root
docker ps -q | xargs -I {} docker inspect {} --format '{{.Name}}: User={{.Config.User}}'
```

- [ ] No containers running as root (User should not be empty or "root")

```bash
# Check no unintended ports are exposed to 0.0.0.0
docker ps --format "{{.Names}}: {{.Ports}}" | grep "0.0.0.0"
```

- [ ] Only Traefik should have 0.0.0.0 bindings (ports 80 and 443)
- [ ] Database containers should NOT have public port bindings

## Fail2ban

```bash
# Check fail2ban status
sudo fail2ban-client status

# Check SSH jail is active
sudo fail2ban-client status sshd
```

- [ ] Fail2ban installed and running
- [ ] SSH jail active
- [ ] Review banned IPs: `sudo fail2ban-client status sshd | grep "Banned IP"`

## Automatic Security Updates

```bash
# Verify unattended-upgrades is configured
cat /etc/apt/apt.conf.d/20auto-upgrades
```

Expected output:
```
APT::Periodic::Update-Package-Lists "1";
APT::Periodic::Unattended-Upgrade "1";
```

- [ ] Automatic security updates enabled

## SSL Certificate Validity

```bash
# Check certificate expiry for your main domain
echo | openssl s_client -servername yourdomain.com -connect yourdomain.com:443 2>/dev/null | openssl x509 -noout -dates
```

- [ ] Certificate valid
- [ ] Not Within 30 days of expiry (Traefik auto-renews, but verify)

## Backup Verification

- [ ] Daily backup n8n workflow is active and executing
- [ ] Last backup run successfully (check n8n execution log)
- [ ] Test restore from backup: restore one backup to a test directory and verify contents

---

## Quarterly Review Additions

Run the above plus:

```bash
# Check for Docker image updates (outdated images may have vulnerabilities)
docker images --format "{{.Repository}}:{{.Tag}}\t{{.CreatedSince}}" | sort

# Update all images
docker compose pull
docker compose up -d
```

- [ ] All base images pulled within last 90 days
- [ ] Check n8n for deprecated node warnings
- [ ] Review active n8n workflows for any that have not executed when expected
- [ ] Review fail2ban banned IP list for unusual patterns
- [ ] Rotate any API keys older than 12 months
